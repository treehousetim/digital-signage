# Project Snapshot — 2026-04-08

State of the `treehousetim/digital-signage` project as of 2026-04-08. Use this as a one-stop briefing for picking the project back up.

## What it is

A standalone HTML/CSS/JS design system for full-screen menu displays. Two pieces:

1. **Renderer** (`renderer.js` + `theme.css`) — embeddable rendering engine that turns a JSON menu definition into a pixel-perfect DOM tree.
2. **Editor** (`editor.js` + `editor.css`) — embeddable three-pane GUI for authoring menus with live preview.

No frameworks, no build step, no npm. Vanilla IIFE pattern. Targets digital signage (TVs, monitors) at 1080p / 2k / 4k in landscape or portrait.

## Data model — top-level shape

```json
{
  "uses":   "themes/warm.json",   // optional: external file imports (string or array)
  "vars":   { "palette": {}, "spacing": {}, "type_scale": {} },
  "theme": {
    "layout":   { "resolution": "4k", "orientation": "landscape",
                  "columns": 1, "column_gutter": "$md", "row_gutter": "$lg",
                  "viewport_padding": {} },
    "colors":   { "background": "$background", ... },
    "fonts":    { "header": {}, "area_title": {}, "item_name": {},
                  "price": {}, "description": {}, /* + custom roles */ },
    "dividers": { "color": "$divider", "width": 1, "style": "solid" },
    "areas":    { /* defaults applied to all areas */ },
    "items":    { /* defaults applied to all items, incl. price_line */ },
    "pricing":  { "symbol": "$", "symbol_position": "before",
                  "symbol_space": false, "format": "full" },
    "header":   { /* visual properties of the header region */ }
  },
  "header": { "elements": [ { "type": "text" | "logo", ... }, ... ] },
  "areas":  [ { "title": "...", "items": [...] }, ... ]
}
```

### Key terminology
- **Vars** — named design values (`palette`, `spacing`, `type_scale`) referenced anywhere with `$name`. Replaces the older "tokens" terminology — never use that word.
- **Font roles** — `theme.fonts` is an open map. Built-in roles are `header`, `area_title`, `item_name`, `price`, `description`. Users can add any role name and reference it from any `font` field. The editor's font dropdowns build dynamically from whatever roles exist.
- **Inheritance cascade** — `default → uses (imported) → theme → area defaults → item override`. Deepest wins.
- **Auto IDs** — areas/items/variations/header elements auto-generate IDs from `title`/`name`/`text` slugs. Both renderer and editor call `autoAssignIds`. Every rendered element gets a `data-ds-id` attribute.

### Spacing units (no back-compat — recently changed)
- `"5%"` — percent of viewport (width for horizontal, height for vertical)
- `"12"` / `"12px"` / `12` — pixels
- `"$name"` — var reference

Built-in `vars.spacing` are now strings: `none='0%'`, `xs='0.25%'`, `sm='0.5%'`, `md='1%'`, `lg='2%'`, `xl='3%'`, `xxl='5%'`.

## Renderer API — `MenuRenderer`

| Method | Description |
|---|---|
| `render(data, target)` | Synchronous render into target DOM. Validates first. |
| `loadFromUrl(url, target)` | Fetch JSON, resolve `uses`, render. Returns Promise. |
| `watch(url, target, intervalSec)` | Polling auto-refresh. Returns `{stop()}`. |
| `validate(data)` | Returns `{valid, errors[], warnings[]}` with JSON paths. |
| `import(url)` | Fetch a single JSON file. Returns `Promise<data>`. |
| `resolve(data, baseUrl)` | Recursively resolve `uses` with cycle detection. Returns `Promise<flatData>`. |
| `autoAssignIds(data)` | Mutate data to ensure all entities have IDs. |
| `formatPrice(value, ctx)` | Format a price string per theme.pricing rules. |

## Editor API — `MenuEditor`

```js
var editor = MenuEditor.create(targetEl, {
  data:        existingMenu,
  onChange:    function (data) {},
  persist:     true,
  storageKey:  'menu-editor:last',
});
editor.getData(); editor.setData(...); editor.destroy();
```

Three resizable panels (tree | inspector | preview). All UI state persists in localStorage:

- `menu-editor:last` — current menu data
- `me-editor:panels` — panel widths
- `me-editor:groups` — collapsed/expanded inspector group state
- `me-editor:selected` — selected tree node path

## Notable features (recently added/changed)

### Layout flattening
`layout` is now a section of `theme` (`theme.layout`), not top-level. `container.columns` / `container.gutter` were flattened to `layout.columns` / `layout.column_gutter`. Vertical spacing between rows of areas is `layout.row_gutter` (NOT `area_gap`).

### Dynamic font roles
`theme.fonts` is an open map. Add any custom role name in `theme.fonts.X` and reference it from any `font` field — including `font_role` selects in the editor's inspector, which build options dynamically at render time.

### Currency options
`theme.pricing` supports `symbol`, `symbol_position` (`before`/`after`), `symbol_space` (insert space), and `format` (`full`/`fewest`).

### Item price line (leader)
`theme.items.price_line` draws a leader between item name and price:

```json
"price_line": {
  "style": "dots",          // none | dots | dashes | solid
  "color": "$muted",
  "thickness": 2,
  "segment_size": 3,        // dot diameter or dash length (px)
  "gap_size": 6,
  "padding_left":  "$xs",
  "padding_right": "$xs"
}
```

Implementation uses `background-image` (radial-gradient for dots, linear-gradient for dashes) with `background-repeat: space no-repeat` so the first/last segment always fits flush against the edges. Vertically centered on the row's cross axis.

### Editor UX changes
- Resizable panes with drag handle, full-screen overlay during drag prevents iframe stealing mouse events
- Live preview re-fits as panels resize (`window.dispatchEvent('resize')` in drag move)
- Tree action buttons float above the row with a 600ms delayed hide; new hover hides previous popovers immediately
- Group collapse state, panel widths, and selected node persist
- Click in preview opens inspector (auto-widens panel if too narrow), scrolls to top, finds areas/items/header elements
- Trash button on inheritable Style Override groups reverts to inherited
- Default-true checkboxes write explicit `false` when unchecked (so renderer sees the change)

## Known sharp edges / things to be aware of

- The editor only knows about layout via `theme.layout` — earlier code still occasionally referenced `data.layout` and broke orientation/resolution preview sizing. All sites have been migrated, but new code touching layout should always go through `theme.layout`.
- `findPathById` requires that store data and rendered data agree on IDs — both editor (on load and on every change) and renderer (on each render) call `autoAssignIds`. Don't skip this.
- `background-repeat: space` is essential to the price-line dots/dashes — using `repeat-x` would clip the last segment.
- Checkbox handling in the inspector: default-true checkboxes must write explicit `false` when unchecked, otherwise the field is removed and the renderer falls back to the default.
- Drag-resize uses a fullscreen `.me-drag-overlay` to keep the iframe from eating mouse events — don't remove it.
